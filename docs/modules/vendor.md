# 供应商管理

## 1. 业务概念

**供应商（Vendor）** — 提供工业品的企业实体，是系统供应链的核心参与者。

**实体关系：**
- 供应商 ↔ 品牌：多对多（一个供应商可代理多个品牌，一个品牌可有多个供应商）
- 供应商 ↔ 分类：多对多（标识供应商的经营范围）
- 供应商 ↔ SPU：多对多（一个供应商可给多个 SPU 供货）
- 供应商 ↔ SKU：多对多（记录每个 SKU 从各供应商的采购/销售价）

**关键枚举字段：**
- 供应商类型：106=原厂、107=授权经销商、236=品牌方、237=总代理、238=其他
- 收款方式、结款方式、付款期限、币种 — 均需查枚举

---

## 2. 前置条件

- 操作前无需特殊前置条件

---

## 3. 判断依据

- **用户只说名称** → 创建基础信息，其他字段填默认值
- **用户说品牌名** → 先查品牌是否存在，不存在则先创建
- **用户说分类** → 先查分类是否存在

---

## 4. 操作流程

### 4.1 供应商开发（从网上搜索到系统入库）

当系统内某个品牌或产品没有供应商时，按以下策略从零开发。**目标：尽可能多地找到合格供应商，不是找一两家就停。**

#### 搜索策略（按优先级，多种渠道都要尝试）

**第一优先：品牌官网授权渠道**

品牌官网是最可靠的来源。找到官网后**必须用 `web_fetch` 打开页面实际查看内容**，不能只看搜索摘要：

- 搜 `{品牌名} 官网` 或 `{品牌英文名} official website` 找到品牌官方网站
- 在官网上找这些页面：**"合作伙伴"、"授权代理"、"渠道查询"、"经销商网络"、"Where to Buy"、"Find a Distributor"、"联系我们"**
- 从官网提取以下信息（按重要性排序）：
  - 邮箱（最优先）— 用于发送询价通知
  - 公司全称、电话、详细地址
  - 联系人姓名及职位（如有）
  - 授权说明/证书描述
  - 官网 URL（记录来源）
- 官网列的授权代理商信息**全部记录**，一家不漏

搜索关键词组合（多试几种）：
- `{品牌名} 中国 授权代理商 经销商 官网`
- `{品牌名} 授权经销商 {地域}`（如"上海 江苏 浙江"）
- `{品牌英文名} China authorized distributor dealer`
- `{品牌英文名} distributor list`

**第二优先：搜索引擎 + B2B 平台**

用多个搜索引擎和 B2B 平台交叉搜索，扩大覆盖范围：

- **搜索引擎**：`{品牌名} 代理商`、`{品牌名} 代理商名录`、`{品牌名} {产品名} 供应商`
- **1688**：搜 `{品牌名} {产品名}`，打开店铺看企业资质
- **百度爱采购**：搜 `{品牌名} 授权代理`
- **icspec.com**、**allchips.com**：电子元器件品牌专用，搜品牌名
- **made-in-china.com**、**gys.cn**：通用工业品平台
- 每次搜到结果**必须用 `web_fetch` 点进去看**，不能只看搜索摘要

**不同搜索词多试**：搜全称找不到 → 搜简称 → 搜英文名 → 搜品牌+产品名 → 搜品牌+供应商 → 搜"品牌名 在哪买" → 搜"品牌名 联系方式"

**第三优先：行业垂直平台**

根据品牌所在行业选择对应平台：
- 工业自动化/气动液压：**工控网(gongkong.com)**、**智能制造网(gkzhan.com)**、**控制工程网(cechina.cn)**、**机电之家(jdzj.com)**
- 电子元器件：**icspec.com**、**allchips.com**、**dianzinav.com**
- 化工仪器：**化工仪器网(chem17.com)**
- 通用：**黄页88(huangye88.com)**、**顺企网(11467.com)**

#### 信息采集标准

每家潜在供应商必须采集：
- 公司全称（用于企查查核验）
- 邮箱或电话（至少有一个，否则无法发询价通知）
- 地址（尽量完整）
- 数据来源链接（记录出处）

#### 企查查核验（每家都要做，不能跳过）

对每家潜在供应商**必须调用以下 MCP 工具**执行核验，不能只靠 WebSearch 摘要：

1. **工商信息** — `mcp__qcc-company__get_company_registration_info(searchKey: "公司全称")`
   → 确认登记状态为「存续」。注销/吊销 → 直接排除
2. **荣誉资质** — `mcp__qcc-operation__get_honor_info(searchKey: "公司全称")`
   → 专精特新、高新技术企业等，记录到供应商 tags
3. **联系方式** — `mcp__qcc-company__get_contact_info(searchKey: "公司全称")`
   → 获取电话、邮箱、官网。必须至少有一个邮箱或电话

核验结果写入供应商 tags（荣誉资质）和 extra-columns（注册资本、信用代码等）。

#### 决策标准

| 条件 | 处理 |
|------|------|
| 存续 + 有联系方式 + 有荣誉 | ✅ 优先创建，全部入库 |
| 存续 + 有联系方式 | ✅ 创建入库 |
| 存续 + 无联系方式 | 尝试其他渠道补，实在没有则排除 |
| 注销/吊销 | ❌ 排除 |
| 重复（同一家公司多个来源） | 合并信息，以最详细的为准 |

> **关键：只要通过核验的，全部创建。不要只留一家——供应商越多，报价竞争越充分。**

#### 创建并绑定

每家用以下命令完成入库：

1. `vendor create` — 创建供应商（先 `--help` 查看参数），tags 填荣誉资质
2. `vendor contact add` — 维护联系人（先 `--help` 查看参数）
3. `vendor bind-brand` — 绑定品牌（先 `--help` 查看参数）
4. `vendor bind-category` — 绑定分类（先 `--help` 查看参数）
5. `vendor super-category add` — 设置优势分类（可选）
6. `vendor get` — 验证创建结果，同时获取跳转链接

系统没有对应字段的信息（如法人、注册资本、品牌代理等级等）用 `--extra-columns` 保存。

#### 业务场景兼容

| 场景 | 搜索策略 |
|------|---------|
| 已知品牌 | 按品牌搜授权代理商 |
| 已知品牌 + 产品名 | 品牌官网 + `{品牌} {产品} 经销商` |
| 已知品牌 + 型号 | `{品牌} {型号} 供应商` + 品牌官网 |
| 只有产品名 | 先确认品牌 → 再按品牌搜 |

### 4.2 查询供应商
→ 列表：`vendor list`（支持名称搜索、分页）
→ 详情：`vendor get`
→ 下拉框：`vendor dropdown`

### 4.3 更新供应商
→ 先 `vendor get` 查看当前值
→ 再 `vendor update`（仅传需要修改的字段）
→ 验证更新结果

### 4.4 删除供应商
→ 先 `vendor get` 查看详情
→ 展示给用户确认（级联清理：供应商-品牌绑定、供应商-分类绑定）
→ 确认后执行 `vendor delete`
→ 验证删除

### 4.5 合并供应商
→ 使用 `vendor merge`
- 将源供应商的数据合并到目标供应商
- 合并选项：品牌、分类、产品

---

## 5. 边界情况

- **供应商已存在** — 按名称精确搜索确认后，展示已有数据给用户
- **品牌/分类不存在** — 提示用户先创建，或用户仅说名称时自动创建
- **删除有绑定关系的供应商** — 级联清理绑定，但不会删除品牌/分类本身
- **枚举值不确定** — 先跑 `enum --help` 查看可用参数后查询枚举值
