# Workflow 02：供应商+产品批量维护

**触发条件**：用户需要批量添加新供应商及其产品（多品牌、多供应商、整套产品上架）。

---

## Phase 1（并行）：准备阶段

由 `wkea-vendor-expert` 和 `wkea-product-expert` 并行执行：

- **`wkea-vendor-expert`**：按用户给的品牌/产品类别搜索授权代理商 → 企查查核验 → 输出供应商候选列表（含联系方式、资质、来源 URL）
- **`wkea-product-expert`**：分析目标产品规格模板，输出 SPU/SKU 结构（规格、属性、单位）

**输出**：供应商列表（待入库）+ 产品模板（待创建）

---

## Phase 2（串行）：创建入库

按 vendor → product 顺序：

1. **`wkea-vendor-expert`**：用 `vendor create` 批量入库供应商，绑定品牌分类
2. **`wkea-product-expert`**：用 `product quick-create` 创建 SPU/SKU，绑定供应信息

**输出**：创建结果汇总（每个供应商的 ID + 每个产品的 SKU ID）

---

## Phase 3：主理人汇编

汇总所有 Phase 2 结果，向用户输出维护报告（含跳转链接、失败原因）。

---

## 异常处理

| 场景 | 处理 |
|------|------|
| 供应商核验不通过 | 跳过该供应商，继续下一个 |
| 产品缺规格数据 | 退回 Phase 1 让 product-expert 补充搜索 |
| 批量创建部分失败 | 已成功的保留，失败的标红给用户确认 |
