# Workflow 06：供应商评估与确认

**触发条件**：
- 用户提供了一组供应商候选名单，要从中确认"今天该把订单给谁"
- 用户要找品牌的授权代理或源头工厂做评估
- 已有候选池，需要按"硬门槛+加权门槛"评 4 档分级
- 业务场景：年度框架协议、大单采购、供应商池清洗

**视角**：供应商（`wkea-vendor-expert` 主配合 + `preferred-supplier-confirm` / `source-supplier-evaluator` 主执行）

**与其他 workflow 的区别**：
- **04**：品牌开发供应商（找新的授权代理）→ vendor-expert 主导，写库
- **06**：供应商评估与确认（在已知池里分级排序）→ preferred/source 主导，不一定写库

---

## 流程边界

**主流程**（同步完成）：
- Phase 1 识别评估类型
- Phase 2 主理人调度对应 expert
- Phase 3 汇编评估报告

**主流程结束点** = Phase 3 输出 HTML 评估报告 + 推荐等级排序。

---

## Phase 1：识别评估类型

主理人判断走哪个子流程：

| 用户意图 | 调度对象 | 方法论 |
|---------|---------|--------|
| **已知名单里选首选**（5 家代理里选 1 家下大单）| `preferred-supplier-confirm` | 硬门槛+加权门槛评分，输出 4 档分级 |
| **找源头工厂**（排除品牌/代理，找 OEM 实际产线）| `source-supplier-evaluator` | 双门槛决策模型 + 产业带识别 |
| **混合**（先找源头或代理，再做分级）| 两个 expert 协作 | vendor-expert 找源 + preferred 做最终分级 |

### 询问用户

如果意图不清，先反问：

> 请问您想要哪种类型的评估？
> 1. 已知名单里挑最佳（推荐首选）
> 2. 找真实生产源头工厂（OEM 厂）
> 3. 两者都要（先找源再分级）
> 4. 只是初步筛选，还没具体目标

## Phase 2：分场景 SOP

### 场景 A：已知名单分级（preferred-supplier-confirm 主导）

适用：业务人员给出 5-10 家候选代理商，问"哪家应该作为今年的主供应商？"

完整 SOP 见 [`preferred-supplier-confirm.md`](../agents/preferred-supplier-confirm.md)。

核心要点：
- **硬门槛**（一票否决）：品牌官网展示名单 + 区域定位
- **加权门槛**（决定档位）：授权等级 + 有效期 + 账期 + 价格表
- 输出 4 档：★★★★★ 首选 / ★★★★ 推荐 / ★★★ 备选 / ★ 不推荐
- 双交付物：Excel + HTML

### 场景 B：源头工厂定位（source-supplier-evaluator 主导）

适用：业务人员要"找真实能生产这种气动元件的厂家，排除品牌和代理"。

完整 SOP 见 [`source-supplier-evaluator.md`](../agents/source-supplier-evaluator.md)。

核心要点：
- **Stage 1 硬门槛**：工信部"百十万千"培育名单 + 产业带匹配度
- **Stage 2 加权门槛**：结算账期 + 价格表折扣 + OEM/中性包装支持
- 候选池反向锁定 5-8 家
- 排除品牌方、授权代理、贸易商，只看实际生产制造商

### 场景 C：混合

适用：先找源头/代理候选，再对候选做最终分级。

主理人调度顺序：先 vendor-expert 找源 → 再 preferred-supplier-confirm 分级。

## Phase 3：汇编评估报告

由主理人执行。

生成 HTML 报告，按 `docs/report-spec.md` 规范，结构：

1. **头部** — 任务类型、生成时间、用户原文
2. **结论摘要** — 首选是谁、备选是谁、哪些不推荐
3. **用户原文** — 用户给出的候选名单或评估需求
4. **评估方法** — 用了哪种评分框架（硬门槛+加权门槛 / 双门槛决策模型）
5. **候选对比表** — 每个供应商的得分明细
6. **首选档案** — 第一名的完整档案（联系方式、关键资产、推荐理由）
7. **跳转链接** — 系统已有供应商的后台链接
8. **待补充/风险** — D3/D4/D5 等需要人工跟进的项
9. **页脚** — 生成时间 + 免责声明

文件命名：
- `/tmp/wkea-supplier-grade-{品牌名}-{日期}.html`（场景 A）
- `/tmp/wkea-source-factory-{产品型号}-{日期}.html`（场景 B）

## Phase 4：流程核验（强制，不可跳过）

由主理人执行：派 `wkea-inspection-expert` 对照 workflow 原文逐步骤核验，生成 HTML 核验报告。**跳过核验 = 流程未完成。**

---

## 异常处理

| 场景 | 处理 |
|------|------|
| 候选池少于 3 家 | 提示用户"建议先补充到至少 5 家再评估" |
| 官网未公示的"一级代理" | 强制标记"不推荐"，不采信第三方平台的宣传 |
| 源头评估混淆了代理 | 重新评估，代理必须降级为"不推荐" |
| 用户想写库 | 评估确认后转 workflow 04 写库 |
| 业务在需求询价流程中 | 把评估结果交给 demand-expert 写 aiRemark 区域 6 |
