# Workflow 03：产品了解与研究

**触发条件**：
- 用户给了产品型号、图片、参数描述，想先了解这个产品
- 用户说"帮我查一下这个产品""这个是什么""有没有这个型号"
- 用户丢了一个或多个产品，没有明确说要创建还是询价

**视角**：产品（`wkea-product-expert` 主导，纯研究不写库）

**与 01 的区别**：
- 01 是**需求驱动**（客户要买 → 研究 → 询价），本 workflow 是**了解驱动**（先搞清楚是什么 → 再决定下一步）
- 本 workflow 不创建产品、不写 aiRemark、不绑定 SKU，只出研究报告

---

## 流程边界

**主流程**（同步完成）：
- Phase 1 信息收集
- Phase 2 逐个产品深度研究
- Phase 3 汇编研究报告

**主流程结束点** = Phase 3 HTML 报告输出 + 反问用户下一步。

**不做什么**：不创建 SPU/SKU、不绑定需求、不开发供应商。研究完再问用户要不要做这些。

---

## Phase 1：信息整理

由 `wkea-product-expert` 执行。

### Step 1.1 提取产品线索

从用户输入中提取每个产品的已知信息：

| 产品 | 已知线索 |
|------|---------|
| 产品 1 | 型号=XXX、图片=有、品牌=未知、数量=未知 |
| 产品 2 | ... |

把缺的字段标出来（品牌未知、规格参数缺失等），这些就是 Phase 2 要补的。

---

## Phase 2：逐个产品深度研究

由 `wkea-product-expert` 执行。**每个产品独立研究，不要合并**。

### 研究步骤（每个产品都走一遍）

```
2.1 系统内搜索
    - ES 搜索型号、缩短型号、去掉分隔符的变体
    - 搜到了 → 记录已有 SKU ID 和价格
    - 没搜到 → 标记"系统无此产品"

2.2 网上搜索品牌
    - 搜 {型号} + 品牌/厂商/producer/manufacturer
    - 搜 {型号} + datasheet/规格书/选型手册
    - 搜 {型号} + 1688/淘宝/爱采购
    - 中英文双关键词（外资品牌用英文搜一轮）

2.3 官网验证
    - 找到品牌后 → 进官网产品中心确认这个型号真实存在
    - 找到 datasheet → 提取关键规格参数

2.4 规格核对（与用户需求对比）
    - 用户给了规格 → 逐条对比，标 ✅/⚠️/❓
    - 用户没给规格 → 列出 datasheet 里的关键参数
    - ⚠️ 标记铁律：哪怕只差一个字也必须标 ⚠️，附差异解释

2.5 B2B 平台交叉验证
    - 1688/淘宝/爱采购搜同型号
    - 记录价格区间、是否有现货、供应商数量
    - 多个平台都搜不到 → 标记"全网信息稀少"

2.6 替代品检查
    - 系统内有没有同规格的其他品牌产品
    - datasheet 里有没有写替代型号
```

### 输出格式（每个产品）

```markdown
## 产品 N：{产品名称/型号}

| 项目 | 内容 |
|------|------|
| 确定品牌 | XXX（来源：官网）或 未知 |
| 确定型号 | XXX |
| 系统已有 | 是（SKU ID: XXX）/ 否 |
| 规格匹配 | ✅ 完全一致 / ⚠️ 差异项 / ❓ 待核实 |
| 价格参考 | ¥XXX-XXX（来源：1688/淘宝）|
| 供应商数量 | 系统 X 家 / 平台约 X 家 |
| 替代品 | 有（XXX）/ 无 |
| 信息来源 | [来源URL列表] |
```

## Phase 3：汇编研究报告

由 `wkea-product-expert` 或主理人执行。

### 报告内容

生成 HTML 报告，风格统一使用 `docs/report-style.html` 的 CSS，内容包含：

1. **研究摘要** — 一共几个产品、研究到什么程度、关键发现
2. **逐个产品详情** — Phase 2 的每个产品研究结果
3. **信息完整度** — 哪些产品信息充分、哪些缺关键数据
4. **下一步建议** — 可以做什么

### 反问用户

报告生成后，**必须反问用户下一步要做什么**：

> 以上是研究结果。请问接下来需要：
> 1. 登记为需求，帮你找供应商询价（推荐）
> 2. 把这些产品上架到系统
> 3. 继续深挖某个产品
> 4. 仅了解，不需要后续操作

### 文件命名

`/tmp/wkea-product-research-{日期}.html`

---

## 异常处理

| 场景 | 处理 |
|------|------|
| 型号全网搜不到 | 标记"信息稀少"，给用户看搜了哪些关键词，问是否换关键词 |
| 搜到多个品牌同名型号 | 列出所有品牌，让用户确认是哪个 |
| 图片里型号模糊 | 列出可能解读，让用户确认 |
| 没有 datasheet | 标记"无官方规格书"，用 B2B 平台参数兜底 |
| 产品系统内已存在 | 直接给 SKU ID + 价格 + 后台链接 |
